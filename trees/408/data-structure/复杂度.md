$\gdef\spaces#1{~ #1 ~}$
$\gdef\O{\mathcal{O}}$

时间复杂度全名渐近时间复杂度, 是算法的时间量度. 描述了执行某个算法所需要的时间如何随着输入数据量的增长而变化. 

$$ T(n) \spaces= \O(f(n)) $$

显然, 常见时间复杂度的序关系是

$$
\begin{aligned}
\O(1) 
&\spaces< \O(\log n) \spaces< \O(n) \spaces< \O(n\log n) \\
&\spaces< \O(n^k)_{k \ge 2}
\spaces< \O(2^n) \spaces< \O(n!)
\end{aligned}
$$

$\textbf{Example.}$ 我们来分析下述 C 程序中 `sum` 函数的时间复杂度. 

```c
int sum(int n) {
    int count = 0;
    for (int i = n; i > 0; i /= 2)
        for (int j = 0; j < i; j++)
            count += 1;
    return count;
}
```

留意 $j < i$ 而 $i$ 是局部变量, 不能将两个 `for` 的复杂度直接相乘. 但我们可以固定 $i$, 有 $i=\frac{n}{2^k}$ $\implies$ $j < \frac{n}{2^k}$. 求和 $ n \sum_{k} \frac{1}{2^k} = 2n$. 故忽略系数时, `sum` 的时间复杂度是 $\O(n)$.
